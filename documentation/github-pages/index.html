How to Make a Static Website using GitHub Pages

If you don’t have money to buy a domain
GitHub Static Site Free Hosting

Step 1 — Create a GitHub Account

If you don’t have a GitHub account yet, you’ll need to make one. GitHub is a website where programmers and students can store code, collaborate, and even host websites for free.

  1. Go to https://github.com
  2. Click Sign Up.
  3. Enter your email, username, and password.
  4. Verify your email by checking your inbox.

Once your account is ready, you can now create repositories (folders where your website files will be stored).

Step 2 — Create a New Repository

A repository (repo) is like a folder on GitHub where your project files are saved. This will contain your HTML, CSS, and other website files.

  1. Click your profile picture (top right) → choose Your repositories.
  2. Click the green New button.
  3. Enter a repository name (example: test).
  4. Choose Public (so anyone can view your site).
  5. (Optional) Check Add a README file to make it easier to edit later.
  6. Click Create repository.

Step 3 — Add Your Website Files

GitHub Pages needs a file called index.html. This file acts as your homepage (the first page that opens when someone visits your website).

  1. Click Add file → Upload files.
  2. Drag and drop your index.html file (and any other files like style.css or images).
  3. Scroll down and click Commit changes (this saves your upload).

If you don’t have an index.html yet, you can create one directly inside GitHub by clicking Add file → Create new file, naming it index.html, and typing some basic HTML.

Step 4 — Enable GitHub Pages

Now that your files are uploaded, let’s publish them as a live website.

  1. Go to your repository and click Settings (top menu).
  2. On the left sidebar, scroll down to Pages.
  3. Under Branch, select main (or master depending on your repo).
  4. Click Save.

After a few seconds, GitHub will generate a link for your site, usually like this:

https://frenzjay.github.io/test/

Step 5 — Visit Your Website

Wait 1–2 minutes, then open the link GitHub provided. Your static website is now live.

Example: If your username is frenzjay and your repository is test, your site will be at:

https://frenzjay.github.io/test/

Extra Tips

  • If you name your repository username.github.io, it becomes your personal website (no subfolder needed).
  • You can update your site anytime by uploading new files or editing directly on GitHub.
  • Use a separate folder for images, CSS, and JavaScript for better organization.
  • If your page doesn’t update right away, refresh or wait a few minutes.
